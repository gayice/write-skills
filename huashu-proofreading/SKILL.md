# Huashu Proofreading — AI味审校Skill

## 何时触发

检测到以下场景时自动加载此 skill：
- 用户说"审校一下"、"降低AI味"、"太AI了"
- 用户说"自然一些"、"有人味"、"帮我改一改"
- humanness_score 输出 composite ≥ 30
- 用户提到"AI检测率高"、"没人味"

## 核心目标

- 降低AI检测率：目标控制在 composite < 30
- 增加人味：让文章有经验感、有温度、像真人在说话
- 保持原风格：不改文章观点，只改表达方式

## 输入数据

此 skill 接收 humanness_score 的 `paragraph_findings` 作为主要输入：

```json
{
  "paragraph_findings": [
    {
      "paragraph_index": 3,
      "text": "在当今AI技术飞速发展的时代，首先我们必须承认...",
      "issues": ["套话连篇", "结构机械"],
      "rewrite_tip": "直接切主题，不绕弯子"
    }
  ],
  "suggestions": [...]
}
```

## 三遍审校流程

### 第一遍：读取 humanness_score 结果

1. 读取 `paragraph_findings`，按问题数量排序（问题多的段落先改）
2. 读取 `suggestions`，按 score 从低到高处理
3. 确定改写优先级

### 第二遍：逐段改写（按6类AI腔）

对每个有问题的段落，按以下清单改写：

#### 1️⃣ 套话连篇
**问题词**：在当今时代、综上所述、不得不、显而易见、相信大家都知道
**改写方向**：直接切主题，不绕弯子

```
❌ 在当今AI技术飞速发展的时代，我们不得不承认...
✅ AI浪潮来了。我用了两周，说说真实感受。
```

#### 2️⃣ AI句式
**问题词**：不是...而是...、一方面...另一方面...、既...又...
**改写方向**：拆成短句，用多样化表达

```
❌ Claude Code不是简单的工具，而是一个智能助手。
✅ Claude Code不只是补全代码。它能理解你想干嘛。
```

#### 3️⃣ 书面词汇
**问题词**：显著提升、进行操作、实现功能、促进发展、发挥了重要作用
**改写方向**：口语化，具体化

```
❌ 通过充分利用Claude Code的能力，可以显著提升开发效率。
✅ 用好Claude Code，开发速度能快不少。我测下来，每项目省3-5小时。
```

#### 4️⃣ 结构机械
**问题词**：首先、其次、最后、几点、主要体现在
**改写方向**：自然叙事，列表适度

```
❌ Claude Code的优势主要体现在以下几个方面：首先，它有强大的上下文理解能力；其次，它支持多种编程语言。
✅ Claude Code最强的地方是什么？理解上下文。你让它改功能，它知道要动哪几个文件，不会漏掉。
```

#### 5️⃣ 态度中立腔
**问题词**：可能、或许、大概、具体情况具体分析
**改写方向**：明确态度，敢下判断

```
❌ Claude Code在某些场景下可能表现较好，但也存在一些局限性。
✅ Claude Code确实好用。但价格贵，个人开发者可能扛不住。
```

#### 6️⃣ 细节缺失
**问题特征**：大词多、具体细节少、"许多"无数字
**改写方向**：加入具体数字、时间、地点、人物

```
❌ 经过一段时间的使用，我发现Claude Code能够有效提升开发效率。
✅ 用了两周。最明显的变化：之前3天的项目，现在1天半搞定。
```

### 第三遍：通读检查

改写完成后，从头到尾读一遍：
- 感觉像真人在说话吗？
- 有没有新引入的AI腔？
- 节奏是否自然？

## 交互模式

### 全自动模式（默认）
用户说"全自动模式"或"直接改"：
- 直接执行改写，不等待确认
- 最多循环3次 humanness_score 复检
- 每次复检后如果分数仍≥30，继续改写

### 交互模式
用户没有说"全自动"：
- 每改完一个段落，展示改写结果
- 等用户确认后再改下一个
- 用户可以跳过某些段落

## 输出格式

```
## 审校完成

**改写前 → 改写后**

[段落3] 套话连篇
❌ 在当今AI技术飞速发展的时代，我们不得不承认...
✅ AI浪潮来了。我用了两周，说说真实感受。

[段落7] 态度中立腔
❌ 这个问题一方面有道理，但另一方面也可能不完全正确。
✅ 我的判断：伊朗在试探美国的底线。

---

humanness_score 复检：composite = 28.5 ✅ < 30 通过
```

## 与 humanness_score 的配合

```
humanness_score --json → paragraph_findings + suggestions
        ↓
huashu-proofreading 逐段改写
        ↓
humanness_score 复检（--tier3）
        ↓
若 composite ≥ 30 → 再循环（最多3次）
若 composite < 30 → 完成，输出 DONE
```

## 注意事项

- **不改观点**：只改表达方式，不改文章的核心论点
- **保持风格**：如果原文是严肃分析体，保持严肃，不要加太多口语
- **适度原则**：不是每个AI腔都要改，humanness_score认为OK的地方不动
- **不编造**：所有改写基于原文，不凭空添加数据或细节
